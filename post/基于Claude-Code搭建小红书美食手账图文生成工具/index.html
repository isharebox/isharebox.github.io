#Agentic Engineering #aiskills

基于Claude Code搭建小红书美食手账图文生成工具

1. 交互进化

Claude Skills 只需要像老板一样下达自然语言指令。

2. 灵活多变

想换个图片风格？只需直接告诉 Claude换个鲜艳点的颜色，它能听懂人话并动态调整。

3. 本地直连

Claude Skills 直接运行在本地终端，生成的图片在你的文件夹里

搭建项目

Agent Skill目录结构设计如下：

bash
xiaohongshu-card-maker/
├── SKILL.md          # 必填：使用说明 + 元数据
├── scripts/          # 必填：可执行代码
├── references/       # 可选：文档资料
└── assets/           # 可选：模板、资源文件
└── output/            # 必填：输出手账图片的目录

基于目录设计，在.claude目录下新建项目文件夹，\skill-project.claude\skills\目录下新建了xiaohongshu-card-maker目录。

1. 编写skill.md。

从下图可知这个skill名叫xiaohongshu-card-master。它的角色是小红书爆款手账专家，它的主要职责是围绕小八来制作美食手账。

这是我编写提示词的思路，你可以基于我的投喂给豆包来扩写：

markdown
1. 接收用户输入的菜名
2. 调用generate.py进行生图操作
图片布局：
- 采用垂直模块化网格，使用手绘直线、虚线或“纸胶带”进行区域分割。
- 关键步骤间使用手写感的箭头或圆点连接，形成清晰的阅读动线。
图片风格：
-现代简约 Ins 风“精装修”手账，融合矢量插画的精准与水彩画风的温馨。
-IP形象固定为小八。
3. 将生成的图片写入output文件夹

2. 编写generate.py

来到.claude/skills/xiaohongshu-card-maker/scripts目录下，编写generate.py代码。

代码的思路，给ai来扩写：

markdown
1. 接收参数：通过命令行接收 Claude 传来的绘画提示词 (--prompt)。
2. 调用接口：将提示词发送给云端绘图 API（代码中配置的 kg-api.cloud），并指定使用 nano-banana-2-4k模型进行生成。
3. 自动下载：拿到生成的图片链接后，自动将其下载并保存到本地的 output 文件夹中，并按时间戳命名。

测试

回到.claude上级目录，在文件路径处输入cmd打开命令提示符，输入claude进入claude code，再输入 有哪些可用的skill，可以看到下图中，生成小红书手账卡片的skill已经被成功添加。

输入命令: 帮我生成制作糖醋里脊的小红书手账卡片

等待几分钟，手账卡片就制作完成了。

来到output文件夹可以看到糖醋里脊的制作手账图文已经存在。

以上就是整个skill构建的完整流程拆解. 仅用两个文件（SKILL.md 和 generate.py）就完成
