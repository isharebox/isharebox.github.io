Claude Code 已经 100% 自己写代码，为什么 Anthropic 还有上百个工程职位空缺？

当看到“AI 100%自己写代码”这样的新闻标题时，感到困惑是完全正常的。但事实的真相远比你想象的更有趣，也更能揭示未来科技公司的运作模式。

简单来说，**“AI写代码”并不等同于“AI取代工程师”，反而在Anthropic这样的前沿公司，它创造了对顶尖工程师更强烈的需求。**

Claude Code负责人Boris Cherny的工作流恰好能解释这一点。他不是在“敲代码”，而是在“指挥”一支由AI智能体组成的舰队。他会在终端**并行运行5个Claude，同时在网页端再运行5-10个**。他的角色从一个“代码书写者”转变为一个“系统架构师和战略指挥官”。

“ 总得有人去 prompt Claude、和客户沟通、与其他团队协调、决定接下来要做什么，工程正在发生变化，优秀的工程师比以往任何时候都更重要。”

虽然 AI 能处理编码的机械部分，但人类在更高层次的功能上仍然不可或缺。

为了更直观地理解这种转变，不妨看看在AI深度介入前后，工程师角色的变化：

| 维度 | 传统工程师 | Anthropic的“AI驾驭者”工程师 (以Boris Cherny为例) | 空缺职位要招的“新工程师” |

| :--- | :--- | :--- | :--- |

| 核心工作 | 亲手编写每一行代码，调试，修复Bug | 定义目标、制定规则、审核结果。让AI智能体去执行具体编码任务 | 不仅会写代码，更要会**设计系统、定义问题、并驾驭AI**高效完成工作 |

| 工作方式 | 线性的：写代码 -> 测试 -> 修复 -> 下一项 | 并行的“舰队指挥官”：同时指挥多个AI智能体处理不同任务，如一个写功能，一个测试，一个写文档 | 能够管理复杂的工作流，将重复性任务自动化，并**协调AI与人类的工作** |

| 能力要求 | 精通编程语言、框架、算法 | 除了基础编码能力，更需要**强大的架构能力、精准的沟通能力和“驯服”AI的创造力** | 拥有丰富的“**支线任务**”（即个人兴趣和跨领域项目），好奇心强，是能贡献于产品、设计和用户反馈的“**通才**” |

| 核心产出 | 自己负责的模块或功能 | 撬动整个团队的产出效能，一个人可以发挥出一个小型工程团队的力量 | 参与到定义下一代人机协作模式的**规则和工具**建设中 |

理解了工程师角色的根本性转变，就不难明白Anthropic为何要逆势大规模招人了。这背后是几股强大的驱动力：

- 🚀 业务爆炸式增长与平台扩张：Claude Code上线仅6个月就创造了近**10亿美元**的年化营收，增长速度惊人。为了支撑如此迅猛的业务扩张，并持续巩固其在AI编程领域的领先地位，公司必须**组建更大规模的团队**来开发新功能、服务海量用户、探索新领域。一个名为Cowork的新工具，其九成代码虽由AI撰写，但整个项目从概念到落地的**关键架构和产品决策，依然由人类团队在短短10天内完成**。

- 🛠️ 打造更强大工具的需要：Anthropic正在构建的是能够理解整个项目上下文、自动设计、生成并测试代码的“**AI工程师**”。开发这种下一代工具本身就是一个极其复杂的工程，需要最优秀的人类工程师来设计其底层架构、训练模型、建立评估体系。从招聘信息看，他们正在寻找能“**设计和维护可扩展的AI驱动开发工具**”的工程师，这正是为了将AI的能力推向新的高度。

- 🎯 对顶尖人才的持续饥渴：Anthropic自成立之初就将自己定位为一家以**安全、可靠的AI系统**为使命的“公共利益公司”。这种强烈的使命感和独特的文化氛围，使其在硅谷人才战中脱颖而出，**招聘速度是人才流失速度的2.68倍**，远超OpenAI、Meta和谷歌。他们提供的软件工程师年薪高达**32万至56万美元**，吸引着那些不仅追求高薪，更渴望在AI前沿领域做出有社会价值贡献的顶尖人才。

总而言之，Anthropic的上百个职位空缺，不是在招聘被AI替代的“码农”，而是在招募能够 “驾驭”AI、定义未来人机协作模式的“新物种工程师”。

Claude Code的创始人Boris Cherny的工作流已经为我们指明了方向：未来的程序员不再是手拿键盘的“打字员”，而是指挥AI舰队的“战略家”。
